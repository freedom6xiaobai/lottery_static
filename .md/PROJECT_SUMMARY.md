JWT 认证系统✅ **JWT 工具（utils/jwt.js）**- 生成 Access Token（2 小时过期）- 生成 Refresh Token（7 天过期）- 验证 Token 有效性- 支持 Token 刷新机制✅ **认证中间件（middleware/auth.js）**- 从请求头提取 Token- 验证 Token 合法性- 支持 "Bearer token" 格式- Token 过期自动检测✅ **认证服务（services/authService.js）**- 用户登录（密码 bcryptjs 加密验证）- 用户注册（新用户自动分配默认权限）- Token 刷新✅ **认证控制器（controllers/authController.js）**- POST /api/auth/login- POST /api/auth/register- POST /api/auth/refresh-token### 3. RBAC 权限系统✅ **权限中间件（middleware/permission.js）**- 细粒度权限检查- 支持单个权限或权限数组- 自动返回 403 Forbidden 错误✅ **权限格式标准化**- `resource:action` 格式- 如：`user:list`、`user:create`、`role:delete`- 易于理解和管理### 4. 用户管理模块✅ **用户服务（services/userService.js）**- getAll() - 获取所有用户- getById(id) - 获取单个用户- create(userData) - 创建用户- update(id, userData) - 更新用户- delete(id) - 删除用户✅ **用户控制器（controllers/userController.js）**- GET /api/users - 获取用户列表- GET /api/users/:id - 获取单个用户- POST /api/users - 创建用户- PUT /api/users/:id - 更新用户- DELETE /api/users/:id - 删除用户✅ **用户路由（routes/user.js）**- 所有端点都配置了权限检查- 支持 CRUD 操作### 5. 角色管理模块✅ **角色服务（services/roleService.js）**- getAll() - 获取所有角色- getById(id) - 获取单个角色- create(roleData) - 创建角色- update(id, roleData) - 更新角色- delete(id) - 删除角色- addPermission(id, permission) - 为角色添加权限- removePermission(id, permission) - 从角色删除权限✅ **角色控制器（controllers/roleController.js）**- GET /api/roles - 获取角色列表- GET /api/roles/:id - 获取单个角色- POST /api/roles - 创建角色- PUT /api/roles/:id - 更新角色- DELETE /api/roles/:id - 删除角色- POST /api/roles/:id/permissions - 添加权限- DELETE /api/roles/:id/permissions - 删除权限✅ **角色路由（routes/role.js）**- 支持权限的动态管理### 6. 菜单管理模块✅ **菜单服务（services/menuService.js）**- getAll() - 获取所有菜单- getByPermissions(permissions) - 根据权限过滤菜单- getById(id) - 获取单个菜单- create(menuData) - 创建菜单- update(id, menuData) - 更新菜单- delete(id) - 删除菜单✅ **菜单控制器（controllers/menuController.js）**- GET /api/menus - 获取所有菜单- GET /api/menus/my-menus - 获取用户可访问菜单- GET /api/menus/:id - 获取单个菜单- POST /api/menus - 创建菜单- PUT /api/menus/:id - 更新菜单- DELETE /api/menus/:id - 删除菜单✅ **菜单路由（routes/menu.js）**- 支持权限过滤（前端动态菜单）### 7. 完整的 API 端点| 模块 | 端点数量 | 权限保护 ||------|--------|--------|| 认证 | 3 个 权限```### 权限字符串```格式: resource:action如: user:create, user:delete, role:list```### Token 流程```登录 → 生成 token → 保存 token → 请求时带 token → 验证 token → 检查权限 → 执行操作```### 中间件栈```auth 中间件 → permission 中间件 → controller → service → 返回数据```---## ❓ 常见问题**Q: 如何修改用户权限？**A: 在 `services/authService.js` 中修改 users 数组中用户的 permissions 字段。**Q: 如何添加新的权限？**A: 权限是自由定义的字符串，在路由中使用 `permission('your:permission')` 即可。**Q: 密码如何加密？**A: 使用 bcryptjs 自动加密，生产环境中密码安全有保障。**Q: Token 过期怎么办？**A: 使用 refreshToken 调用 `/api/auth/refresh-token` 获取新的 accessToken。**Q: 如何集成数据库？**A: 参考 `ARCHITECTURE.md` 中的"集成数据库"章节，使用 Prisma + MySQL。---## 📞 需要帮助？1. 查看 `QUICKSTART.md` 快速开始2. 阅读 `API_DOCUMENTATION.md` 了解 API 细节3. 研究 `ARCHITECTURE.md` 理解系统设计4. 使用 Postman 测试 API 功能---## 📝
