JWT 工具（utils/jwt.js）**职责:**- 生成 access token（2 小时过期）- 生成 refresh token（7 天过期）- 验证 access token- 验证 refresh token**关键方法:**```javascriptsignAccessToken(payload)      // 生成 access tokensignRefreshToken(payload)     // 生成 refresh tokenverifyAccessToken(token)      // 验证 access tokenverifyRefreshToken(token)     // 验证 refresh token```**流程:**```用户登录 → 提取用户信息 → 签名生成 token → 返回给前端       前端请求 → 验证 token 有效性 → 解码获取用户信息 → 继续处理```### 2. 认证中间件（middleware/auth.js）**职责:**- 从请求头提取 token- 验证 token 是否有效- 将解码后的用户信息存储在 req.user**使用方式:**```javascriptrouter.get('/users', auth, controller.getList);// 如果 token 无效，返回 401 错误// 如果 token 有效，req.user 包含用户信息，继续执行```**支持格式:**```Authorization: Bearer <token>或Authorization: <token>```### 3. 权限中间件（middleware/permission.js）**职责:**- 检查用户是否拥有所需权限- 支持单个权限或权限数组**使用方式:**```javascript// 单个权限router.post('/users', auth, permission('user:create'), controller.create);// 多个权限（满足任一即可）router.delete('/users/:id', auth, permission(['user:delete', 'admin']), controller.delete);```**权限检查流程:**```获取 req.user.permissions   ↓检查是否包含所需权限   ↓包含 → 继续执行不包含 → 返回 403 Forbidden```### 4. 认证服务（services/authService.js）**职责:**- 用户登录逻辑（验证凭证、生成 token）- 用户注册逻辑（密码加密、创建用户）- Token 刷新逻辑**关键方法:**```javascriptlogin(username, password)           // 验证用户，生成 tokenregister(username, password)        // 注册新用户refreshToken(refreshToken)          // 使用 refresh token 获取新的 access token```**密码处理:**```用户输入密码 → bcrypt.hash() 加密存储登录时 → bcrypt.compare() 验证```### 5. 用户服务（services/userService.js）**职责:**- 用户数据的增删改查- 用户数据管理（不含密码）**关键方法:**```javascriptgetAll()                    // 获取所有用户getById(id)                 // 按 ID 获取用户create(userData)            // 创建用户update(id, userData)        // 更新用户delete(id)                  // 删除用户```### 6. 角色服务（services/roleService.js）**职责:**- 角色数据的增删改查- 权限分配和移除**关键方法:**```javascriptgetAll()                         // 获取所有角色getById(id)                      // 按 ID 获取角色create(roleData)                 // 创建角色update(id, roleData)             // 更新角色delete(id)                       // 删除角色addPermission(id, permission)    // 为角色添加权限removePermission(id, permission) // 从角色移除权限```### 7. 菜单服务（services/menuService.js）**职责:**- 菜单数据的增删改查- 根据权限过滤菜单**关键方法:**```javascriptgetAll()                           // 获取所有菜单getByPermissions(permissions)      // 根据权限过滤菜单getById(id)                        // 按 ID 获取菜单create(menuData)                   // 创建菜单update(id, menuData)               // 更新菜单delete(id)                         // 删除菜单```**前端动态菜单生成:**```1. 用户登录获取 token2. 请求 GET /api/menus/my-menus3. 后端根据用户权限返回过滤后的菜单4. 前端基于返回的菜单数据渲染导航菜单```---## 数据模型### User（用户）```javascript{  id: Number,              // 用户 ID  username: String,        // 用户名（唯一）  password: String,        // 加密后的密码  permissions: 错误详情（可选）}```### 常见错误码| 码 | 含义 | 原因 ||----|------|------|| 200 | OK | 请求成功 || 201 | Created | 资源创建成功 || 400 | Bad Request | 请求参数错误 || 401 | Unauthorized | Token 无效或过期 || 403 | Forbidden | 权限不足 || 404 | Not Found | 资源不存在 || 500 | Server Error | 服务器错误 |### 错误处理流程```检测到错误   ↓判断错误类型   ↓返回相应的错误响应   ↓前端捕获错误   ↓根据错误码采取相应行动   ├─ 401: 重新登录或刷新 token   ├─ 403: 提示权限不足   ├─ 404: 提示资源不存在   └─ 500: 提示服务器错误```---## 安全机制### 1. 密码安全```• 使用 bcryptjs 加密存储密码• 加密强度: 10 轮（bcrypt.hash(..., 10)）• 登录时使用 bcrypt.compare() 验证• 密码不在任何 API 响应中返回```### 2. Token 安全```• JWT 使用 HS256 算法签名• 使用环境变量配置 SECRET（生产环境必须修改）• Access Token 短期有效（2 小时）• Refresh Token 长期有效（7 天）• Token 应仅在 HTTPS 连接上传输（生产环境）```### 3. 权限验证```• 每个 API 都明确定义所需权限• 权限检查在中间件中执行（在业务逻辑之前）• 权限检查是细粒度的（resource:action）• 支持权限组合检查```### 4. 输入验证```• 登录: 检查 username 和 password 是否存在• 注册: 检查密码长度（最少 6 字符）• 创建资源: 检查必填字段• 防止 SQL 注入（使用 ORM 时自动防护）```---##
