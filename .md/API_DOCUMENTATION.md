RBAC (Role-Based Access Control) 基于角色的访问控制- **User（用户）**: 系统使用者- **Role（角色）**: 权限的集合（如 admin、user）- **Permission（权限）**: 操作权限（如 user:list、user:create）- **Menu（菜单）**: 前端导航菜单，与权限关联### 2. JWT 认证流程```用户登录   ↓验证用户名和密码  ↓生成 Access Token 和 Refresh Token  ↓返回给前端  ↓前端在请求头中携带 Token```### 3. 权限检查流程```请求 → Auth 中间件（验证 token）→ Permission 中间件（检查权限）→ Controller → Service → 返回数据```---## API 端点详解### 一、认证接口#### 1.1 用户登录**请求:**```bashPOST /api/auth/loginContent-Type: application/json{  "username": "admin",  "password": "admin123"}```**响应:**```json{  "success": true,  "code": 200,  "data": {    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",    "user": {      "id": 1,      "username": "admin",      "permissions": ["user:list", "user:create", "user:update", "user:delete", "role:list", "role:create", "menu:list"]    }  }}```**说明:**- 密码: admin123（账户 admin）或 user123（账户 user）- Access Token 有效期: 2小时- Refresh Token 有效期: 7天---#### 1.2 用户注册**请求:**```bashPOST /api/auth/registerContent-Type: application/json{  "username": "newuser",  "password": "password123"}```**响应:**```json{  "success": true,  "code": 201,  "data": {    "id": 3,    "username": "newuser",    "permissions": ["menu:list"]  }}```**说明:**- 密码最少 6 个字符- 默认权限: menu:list---#### 1.3 刷新 Token**请求:**```bashPOST /api/auth/refresh-tokenContent-Type: application/json{  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}```**响应:**```json{  "success": true,  "code": 200,  "data": {    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."  }}```---### 二、用户管理接口#### 2.1 获取用户列表**权限要求:** user:list**请求:**```bashGET /api/usersAuthorization: Bearer <accessToken>```**响应:**```json{  "success": true,  "code": 200,  "data": [    {      "id": 1,      "username": "admin",      "email": "admin@example.com",      "status": "active"    },    {      "id": 2,      "username": "user",      "email": "user@example.com",      "status": "active"    }  ]}```---#### 2.2 获取单个用户**权限要求:** user:list**请求:**```bashGET /api/users/:idAuthorization: Bearer <accessToken>```**响应:**```json{  "success": true,  "code": 200,  "data": {    "id": 1,    "username": "admin",    "email": "admin@example.com",    "status": "active"  }}```---#### 2.3 创建用户**权限要求:** user:create**请求:**```bashPOST /api/usersAuthorization: Bearer <accessToken>Content-Type: application/json{  "username": "newuser",  "email": "newuser@example.com",  "status": "active"}```**响应:**```json{  "success": true,  "code": 201,  "data": {    "id": 3,    "username": "newuser",    "email": "newuser@example.com",    "status": "active"  }}```---#### 2.4 更新用户**权限要求:** user:update**请求:**```bashPUT /api/users/:idAuthorization: Bearer <accessToken>Content-Type: application/json{  "email": "newemail@example.com",  "status": "inactive"}```**响应:**```json{  "success": true,  "code": 200,  "data": {    "id": 1,    "username": "admin",    "email": "newemail@example.com",    "status": "inactive"  }}```---#### 2.5 删除用户**权限要求:** user:delete**请求:**```bashDELETE /api/users/:idAuthorization: Bearer <accessToken>```**响应:**```json{  "success": true,  "code": 200,  "data": {    "id": 1,    "username": "admin",    "email": "admin@example.com",    "status": "active"  }}```---### POST /api/auth/register   ├─ 创建新用户   ├─ 默认权限: [menu:list]   2. POST /api/auth/login   ├─ 获取 accessToken 和 refreshToken   3. GET /api/menus/my-menus (带 accessToken)   ├─ 返回该用户可访问的菜单   └─ 前端基于此动态渲染导航```### 场景 2: 管理员创建新角色并分配权限```1. GET /api/roles (验证管理员权限)   ├─ 获取现有角色列表   2. POST /api/roles   ├─ 创建新角色（带基础权限）   3. POST /api/roles/:id/permissions   ├─ 为该角色添加权限   4. 新用户分配该角色后自动获得这些权限```### 场景 3: 权限不足时的处理```1. GET /api/users (无 user:list 权限)   2. 中间件返回:   {     "success": false,     "code": 403,     "message": "Forbidden - insufficient permissions"   }   3. 前端捕获 403 错误，提示用户权限不足```---## 默认用户凭证### Admin 用户- **用户名:** admin- **密码:** admin123- **权限:** user:list, user:create, user:update, user:delete, role:list, role:create, menu:list### Regular 用户- **用户名:** user- **密码:** user123- **权限:** user:list, menu:list---## 错误处理### 401 Unauthorized（无 Token 或 Token 失效）```json{  "success": false,  "code": 401,  "message": "No token provided" 或 "Invalid token" 或 "Token expired"}```**处理方案:**- 前端检测到 401 错误- 尝试使用 refreshToken 获取新的 accessToken- 如果 refreshToken 也失效，跳转到登录页### 403 Forbidden（权限不足）```json{  "success": false,  "code": 403,  "message": "Forbidden - insufficient permissions"}```**处理方案:**- 提示用户没有操作权限- 可显示可访问的功能列表### 404 Not Found（资源不存在）```json{  "success": false,  "code": 404,  "message": "User not found" 或 "Role not found"}```### 400 Bad Request（参数验证失败）```json{  "success": false,  "code": 400,  "message": "Username and password are required"}```---## 进阶集成### 推荐下一步1. **数据库集成**   - 使用 Prisma + MySQL 替代内存存储   - 创建 user、role、permission、menu 表2. **前端集成**   - Vue 3 或 React   - 动态菜单和权限渲染   - Token 自动刷新机制3. **生产级增强**   - 操作日志记录   - API 限流   - 密码强度验证   - 2FA 认证4. **监控和部署**   - Docker 容器化   - PM2 进程管理   - 错误追踪（Sentry）   - 性能监控---## 技术栈- **后端框架:** Express.js- **认证:** JWT (jsonwebtoken)- **密码加密:** bcryptjs- **ORM:** 可选（Prisma + MySQL）- **日志:** morgan- **版本管理:** npm---## 相关文件- `middleware/auth.js` - JWT 验证中间件- `middleware/permission.js` - RBAC 权限检查中间件- `utils/jwt.js` - JWT 签名和验证工具- `services/authService.js` - 认证业务逻辑- `services/userService.js` - 用户管理业务逻辑- `services/roleService.js` - 角色管理业务逻辑- `services/menuService.js` - 菜单业务逻辑- `controllers/*Controller.js` - 各模块控制器- `routes/*.js` - API 路由定义
