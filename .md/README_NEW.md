# Admin RBAC 系统完整的 Node.js + Express 后台管理系统，包含 **RBAC（基于角色的访问控制）** 实现。## ✨ 核心特性- ✅ **JWT 认证** - Access Token + Refresh Token 双 Token 机制- ✅ **RBAC 权限系统** - 灵活的基于角色的权限控制- ✅ **用户管理** - 完整的用户 CRUD 操作- ✅ **角色管理** - 角色创建、权限分配、权限撤销- ✅ **菜单管理** - 动态菜单、权限过滤- ✅ **密码加密** - bcryptjs 加密存储- ✅ **三层架构** - routes → controllers → services- ✅ **完整文档** - API 文档、快速开始、架构说明## 🚀 快速开始### 1. 安装依赖```bashnpm install```### 2. 启动服务```bashnpm start```服务将运行在 `http://localhost:3000`### 3. 测试 API使用 Postman 导入 `postman_collection.json`，或查看 `API_DOCUMENTATION.md` 了解所有端点。## 📚 文档指南| 文档 | 内容 | 适合场景 ||------|------|---------|| [QUICKSTART.md](./QUICKSTART.md) | 快速开始、默认用户、测试流程 | 快速上手 || [API_DOCUMENTATION.md](./API_DOCUMENTATION.md) | 所有 21 个 API 端点详细说明 | API 集成 || [ARCHITECTURE.md](./ARCHITECTURE.md) | 系统架构、数据模型、扩展指南 | 深入理解 || [PROJECT_SUMMARY.md](./PROJECT_SUMMARY.md) | 项目完成总结 | 功能总览 |## 🔑 默认用户### Admin 账户- 用户名: `admin`- 密码: `admin123`- 权限: 所有权限### Regular 账户- 用户名: `user`- 密码: `user123`- 权限: user:list, menu:list## 📊 API 端点总览### 认证 (3 个)- `POST /api/auth/login` - 登录- `POST /api/auth/register` - 注册- `POST /api/auth/refresh-token` - 刷新 Token### 用户管理 (5 个)- `GET /api/users` - 获取用户列表- `GET /api/users/:id` - 获取单个用户- `POST /api/users` - 创建用户- `PUT /api/users/:id` - 更新用户- `DELETE /api/users/:id` - 删除用户### 角色管理 (7 个)- `GET /api/roles` - 获取角色列表- `GET /api/roles/:id` - 获取单个角色- `POST /api/roles` - 创建角色- `PUT /api/roles/:id` - 更新角色- `DELETE /api/roles/:id` - 删除角色- `POST /api/roles/:id/permissions` - 添加权限- `DELETE /api/roles/:id/permissions` - 删除权限### 菜单管理 (6 个)- `GET /api/menus` - 获取所有菜单- `GET /api/menus/my-menus` - 获取我的菜单（权限过滤）- `GET /api/menus/:id` - 获取单个菜单- `POST /api/menus` - 创建菜单- `PUT /api/menus/:id` - 更新菜单- `DELETE /api/menus/:id` - 删除菜单## 📁 项目结构```lottery_static/├── middleware/               # 中间件层│   ├── auth.js              # JWT 认证中间件│   └── permission.js        # RBAC 权限检查中间件├── utils/│   └── jwt.js               # JWT 工具函数├── controllers/             # 控制器层│   ├── authController.js│   ├── userController.js│   ├── roleController.js│   └── menuController.js├── services/                # 业务逻辑层│   ├── authService.js│   ├── userService.js│   ├── roleService.js│   └── menuService.js├── routes/                  # 路由层│   ├── auth.js│   ├── user.js│   ├── role.js│   └── menu.js├── app.js                   # Express 主文件├── package.json├── .env.example             # 环境变量示例└── 文档文件    ├── QUICKSTART.md    ├── API_DOCUMENTATION.md    ├── ARCHITECTURE.md    └── PROJECT_SUMMARY.md```## 🔒 安全特性- **密码加密**: bcryptjs 加密存储，支持 bcrypt.compare() 验证- **Token 安全**: JWT HS256 签名，支持 Token 刷新机制- **权限验证**: 请求级别的权限检查，细粒度权限控制- **输入验证**: 参数存在性检查，密码强度验证## 🛠️ 技术栈- **运行环境**: Node.js 12+- **Web 框架**: Express.js- **认证**: JWT (jsonwebtoken)- **密码加密**: bcryptjs- **日志**: morgan- **模板**: pug## 📝 权限字符串格式所有权限遵循 `resource:action` 格式：```user:list      - 查看用户列表user:create    - 创建用户user:update    - 更新用户user:delete    - 删除用户role:list      - 查看角色列表role:create    - 创建角色role:update    - 更新角色role:delete    - 删除角色menu:list      - 查看菜单menu:create    - 创建菜单menu:update    - 更新菜单menu:delete    - 删除菜单```## 🔄 常见业务流程### 用户登录流程```1. POST /api/auth/login   ├─ 验证用户名和密码   ├─ 生成 Access Token (2h)   ├─ 生成 Refresh Token (7d)   └─ 返回 token 和用户信息2. 前端保存 token   ├─ 将 token 存储在 localStorage 或 sessionStorage   └─ 后续请求在 Authorization 头中携带3. 前端请求受保护资源   ├─ Authorization: Bearer <token>   └─ 服务器验证 token 有效性和权限```### 动态获取用户菜单流程```1. 用户登录获取 token   └─ token 包含用户的所有权限信息2. 前端调用 GET /api/menus/my-menus   ├─ 携带 Authorization: Bearer <token>   └─ 服务器根据用户权限过滤菜单3. 后端返回用户可访问的菜单列表   └─ 前端基于此动态渲染导航菜单```## 🧪 测试 API### 使用 Postman1. 导入 `postman_collection.json`2. 调用 Login 接口获取 token3. 将 token 设置到 Postman 📖 主要模块说明### JWT 工具 (utils/jwt.js)生成和验证 JWT token，支持两种 token 类型。### 认证中间件 (middleware/auth.js)验证请求中的 token 有效性，从 token 中提取用户信息。### 权限中间件 (middleware/permission.js)检查用户是否拥有特定权限，支持权限组合。### 服务层 (services/*)包含所有业务逻辑，与数据库交互（当前使用内存存储）。### 控制器层 (controllers/*)处理 HTTP 请求和响应，调用服务层。### 路由层 (routes/*)定义 API 端点和中间件栈。## 💻 开发说明### 添加新的权限在路由中使用新的权限字符串即可：```javascriptrouter.post('/articles', auth, permission('article:create'), controller.create);```### 修改用户权限在 `services/authService.js` 中编辑 users 数组的 permissions 字段。### 修改 Token 过期时间在 `utils/jwt.js` 中修改 `expiresIn` 参数。## ❓ 常见问题**Q: 如何修改默认 JWT_SECRET？**A: 在 `.env` 文件中修改 JWT_SECRET 和 JWT_REFRESH_SECRET。**Q: Token 过期后怎么办？**A: 使用 refreshToken 调用
