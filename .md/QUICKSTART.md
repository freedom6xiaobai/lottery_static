# 快速开始指南## 项目概述这是一个完整的 Node.js + Express Admin 后台 RBAC（基于角色的访问控制）系统实现。包含用户认证、角色管理、权限控制、菜单管理等核心功能。---## 项目已完成的功能✅ **JWT 认证**- 用户登录和注册- Access Token 和 Refresh Token 支持- 密码使用 bcryptjs 加密✅ **RBAC 权限控制**- 基于角色的权限管理- 灵活的权限检查中间件- 支持多权限和单权限检查✅ **完整的 API 端点**- 认证接口（登录、注册、刷新 Token）- 用户管理（CRUD）- 角色管理（CRUD + 权限分配）- 菜单管理（CRUD + 权限过滤）✅ **三层架构**- `routes/` - 路由层- `controllers/` - 控制器层- `services/` - 业务逻辑层- `middleware/` - 中间件层- `utils/` - 工具层---## 安装和运行### 1. 安装依赖所需依赖已经安装，如需重新安装：```bashnpm install```### 2. 配置环境变量创建 `.env` 文件（可复制 `.env.example`）：```bashcp .env.example .env```编辑 `.env` 文件，修改以下配置：```envJWT_SECRET=your-secret-key-change-in-productionJWT_REFRESH_SECRET=your-refresh-secret-key-change-in-productionPORT=3000NODE_ENV=development```### 3. 启动服务器```bashnpm start```或使用开发模式（自动重启）：```bashnpm install -g nodemon  # 如需要nodemon```服务器将在 `http://localhost:3000` 启动---## 默认用户系统预置两个用户，可直接用于测试：### Admin 用户```用户名: admin密码: admin123权限: user:list, 核心概念理解### RBAC 流程图```┌─────────────┐│   用户登录   ││username/pwd │└──────┬──────┘       │       ▼┌──────────────────┐│   验证凭证        ││ bcrypt 校验密码  │└──────┬───────────┘       │       ▼┌──────────────────────┐│  生成 JWT Token      ││ + 用户权限信息       │└──────┬───────────────┘       │       ▼┌──────────────────────┐│  返回 accessToken    ││  + refreshToken      │└──────────────────────┘       │       │ 前端保存 token，后续请求带在 Authorization 头中       │       ▼┌──────────────────────┐│  请求受保护资源       ││ Authorization: ...   │└──────┬───────────────┘       │       ▼┌──────────────────────┐│  Auth 中间件         ││ 验证 token 有效性    │└──────┬───────────────┘       │       ▼┌──────────────────────┐│  Permission 中间件    ││ 检查用户权限         │└──────┬───────────────┘       │       ▼┌──────────────────────┐│  Controller          ││ 处理业务逻辑         │└──────┬───────────────┘       │       ▼┌──────────────────────┐│  Service            ││ 数据操作和处理       │└──────┬───────────────┘       │       ▼┌──────────────────────┐│  返回结果            ││ { success, data }    │└──────────────────────┘```### 权限字符串格式所有权限遵循 `resource:action` 格式：- `user:list` - 查看用户列表- `user:create` - 创建用户- `user:update` - 更新用户- `user:delete` - 删除用户- `role:list` - 查看角色列表- `role:create` - 创建角色- `role:update` - 更新角色- `role:delete` - 删除角色- `menu:list` - 查看菜单- `menu:create` - 创建菜单- `menu:update` - 更新菜单- `menu:delete` - 删除菜单---## 常见测试场景### 场景 1: 测试权限不足```bash# 用 user 账户登录（只有 user:list 和 menu:list 权限）curl -X POST http://localhost:3000/api/auth/login \  -H "Content-Type: application/json" \  -d '{"username":"user","password":"user123"}'# 尝试创建用户（需要 user:create 权限，user 账户没有）curl -X POST http://localhost:3000/api/users \  -H "Authorization: Bearer <user-token>" \  -H "Content-Type: application/json" \  -d '{"username":"newuser","email":"new@example.com"}'# 返回 403 Forbidden# {#   "success": false,#   "code": 403,#   "message": "Forbidden - insufficient permissions"# }```### 场景 2: Token 过期刷新```bash# 当 accessToken 过期，使用 refreshToken 获取新的 accessTokencurl -X POST http://localhost:3000/api/auth/refresh-token \  -H "Content-Type: application/json" \  -d '{"refreshToken":"<refresh-token>"}'# 返回新的 accessToken# {#   "success": true,#   "code": 200,#   "data": {#     "accessToken": "new-token..."#   }# }```### 场景 3: 动态获取用户菜单```bash# 用不同权限的用户登录，获取 token# Admin 用户curl -X GET http://localhost:3000/api/menus/my-menus \  -H "Authorization: Bearer <admin-token>"# 返回所有菜单（因为 admin 有所有权限）# Regular 用户curl -X GET http://localhost:3000/api/menus/my-menus \  -H "Authorization: Bearer <user-token>"# 返回部分菜单（只有权限允许的菜单）```---## auth, userController.getList);```### permission.js - 权限检查中间件```javascript// 作用: 在 auth 中间件之后使用，检查用户是否有特定权限// 支持单个权限或权限数组// 使用方式router.post('/users', auth, permission('user:create'), userController.create);router.delete('/users/:id', auth, permission(['user:delete', 'admin']), userController.delete);```---## 下一步建议### 短期（必做）1. **集成数据库**   - 使用 Prisma + MySQL   - return jwt.sign(payload, SECRET, { expiresIn: '1h' });};```### Q2: 如何添加新的权限？**A:** 权限是字符串，可以随意添加。例如在 `services/roleService.js` 中修改角色权限：```javascript{  id: 3,  name: 'editor',  permissions: ['article:list', 'article:create', 'article:update', 'comment:delete']}```然后在路由中使用该权限：```javascriptrouter.post('/articles', auth, permission('article:create'), controller.create);```### Q3: 如何处理 Token 过期？**A:** 前端应在收到 401 错误时，使用 refreshToken 获取新的 accessToken：```javascript// 如果 401 错误if (error.response.status === 401) {  const newToken = await refreshAccessToken(refreshToken);  // 重试原请求}```### Q4: 密码是否加密存储？**A:** 是的。使用 bcryptjs 加密：```javascript// 注册时const hashedPassword = await bcrypt.hash(password, 10);// 登录时验证const valid = await bcrypt.compare(password, hashedPassword);```### Q5: 如何添加新的用户？**A:** 由于当前使用内存存储，重启服务器后会重置。添加用户有两种方式：1. **通过 API 注册:**```bashPOST /api/auth/register{"username":"newuser","password":"password123"}```2. **直接修改 authService.js 中的 users 数组**---## 问题排查### 错误: "No token provided"**原因:** 请求头中没有 Authorization**解决:** 确保请求包含：```Authorization: Bearer <token>```### 错误: "Invalid token"**原因:** token 格式错误或已过期**解决:** 1. 检查 token 是否完整2. 使用 refreshToken 获取新的 accessToken### 错误: "Forbidden - insufficient permissions"**原因:** 用户没有该操作的权限**解决:** 1. 检查用户的权限列表2. 由管理员分配该权限3. 或使用权限更高的账户---##
