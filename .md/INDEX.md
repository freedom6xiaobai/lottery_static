Refresh Token**middleware/auth.js**- 验证请求中的 Token- 提取用户信息到 req.user**services/authService.js**- 用户登录逻辑- 密码加密和验证- Token 生成**controllers/authController.js**- 处理 HTTP 请求- 返回登录结果**routes/auth.js**- 定义认证路由- 映射到控制器### 权限检查关键文件**middleware/permission.js**- 检查用户是否有特定权限- 返回 403 Forbidden 如果权限不足**services/roleService.js**- 管理角色和权限- 添加/删除权限**services/menuService.js**- 根据权限过滤菜单- 用于前端动态菜单---## 📊 API 端点快速查询### 认证- `POST /api/auth/login` - 登录- `POST /api/auth/register` - 注册- `POST /api/auth/refresh-token` - 刷新 Token### 用户（需要登录）- `GET /api/users` - 获取列表（需要 user:list）- `POST /api/users` - 创建（需要 user:create）- `PUT /api/users/:id` - 更新（需要 user:update）- `DELETE /api/users/:id` - 删除（需要 user:delete）### 角色（需要登录）- `GET /api/roles` - 获取列表（需要 role:list）- `POST /api/roles` - 创建（需要 role:create）- `PUT /api/roles/:id` - 更新（需要 role:update）- `POST /api/roles/:id/permissions` - 添加权限### 菜单（需要登录）- `GET /api/menus` - 获取全部（需要 menu:list）- `GET /api/menus/my-menus` - 获取我的菜单- `POST /api/menus` - 创建（需要 menu:create）---## 🔐 默认用户```用户 1：  账户: admin / admin123  权限: 所有权限用户 2：  账户: user / user123  权限:
