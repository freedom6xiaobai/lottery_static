# 🚀 数据库集成 - 快速参考## ⚡ 5 分钟快速开始```bash# 1️⃣ 编辑 .env（填入你的数据库 URL）# DATABASE_URL=postgresql://user:password@localhost:5432/lottery_db# 2️⃣ 初始化数据库npm run db:generate    # 生成 Prisma 客户端npm run db:migrate     # 创建所有表npm run db:seed        # 填充初始数据# 3️⃣ 启动服务npm start```---## 📋 新增文件总结### 核心文件| 文件 | 说明 ||------|------|| `prisma/schema.prisma` | 数据库模型定义 || `prisma/seed.js` | 初始化脚本 || `lib/db.js` | Prisma 连接管理 |### 数据库服务（替代内存版本）| 文件 | 说明 ||------|------|| `services/authService_db.js` | 认证（数据库版） || `services/userService_db.js` | 用户管理（数据库版） || `services/roleService_db.js` | 角色管理（数据库版） || `services/menuService_db.js` | 菜单管理（数据库版） |### 文档| 文件 | 说明 ||------|------|| `DATABASE_GUIDE.md` | 完整数据库指南 || `MIGRATION_GUIDE.md` | 迁移指南 |---## 🔧 常用命令```bashnpm run db:generate        # 生成 Prisma 客户端npm run db:migrate         # 运行迁移（创建表）npm run db:seed            # 填充初始数据npm run db:studio          # 打开 Prisma GUI（http://localhost:5555）npm run db:reset           # 重置数据库（⚠️ 删除所有数据）npm run db:push            # 推送模型（无需迁移文件）npm start                  # 启动服务（生产）npm run dev                # 启动服务（开发，自动重启）```---## 🗄️ 数据库配置### PostgreSQL（推荐）```envDATABASE_URL=postgresql://user:password@localhost:5432/lottery_db?schema=public```### MySQL```envDATABASE_URL=mysql://root:password@localhost:3306/lottery_db```---## 📝 代码迁移修改 4 个 controller 文件中的 require 语句：```javascript// 原来（内存版本）const authService = require('../services/authService');// 改为（数据库版本）const authService = require('../services/authService_db');```对这 4 个文件做相同修改：- `controllers/authController.js`- `controllers/userController.js`- `controllers/roleController.js`- `controllers/menuController.js`---## 🎯 初始化数据运行 `npm run db:seed` 会自动创建：| 类型 | 数量 | 示例 ||------|------|------|| 权限 | 12 | user:list, user:create, ... || 角色 | 2 | admin, user || 用户 | 2 | admin/admin123, user/user123 || 菜单 | 4 | User Management, Dashboard, ... |---## ✅ 验证数据库### 方法 1：使用 Prisma Studio```bashnpm run db:studio# 打开浏览器访问 http://localhost:5555```### 方法 2：测试 API```bashnpm start# 在另一个终端运行curl -X POST http://localhost:3000/api/auth/login \  -H "Content-Type: application/json" \  -d '{"username":"admin","password":"admin123"}'```---## 🚨 故障排查| 错误 | 原因 | 解决方案 ||------|------|--------|| `ECONNREFUSED` | 数据库未运行 | `brew services start postgresql` || `permission denied` | 权限不足 | 检查数据库用户权限 || `Module not found` | Prisma 未生成 | `npm run db:generate` || `Migration failed` | 迁移冲突 | `npm run db:reset` |---## 📚 详细文档- **DATABASE_GUIDE.md** - 完整的数据库指南（2000+ 行）- **MIGRATION_GUIDE.md** - 从内存迁移到数据库（500+ 行）---## 🔄 下次启动```bash# 只需一条命令即可启动（数据库已初始化）npm start```所有数据都会自动从数据库读取和保存！---**现在开始使用数据库吧！** 🎉
