# 🔄 数据库迁移 - 从内存到数据库## 快速开始（5 分钟）### 1️⃣ 确保 PostgreSQL 运行中```bash# macOSbrew services start postgresql# 或创建本地数据库createdb lottery_db```### 2️⃣ 配置数据库连接编辑 `.env` 文件：```envDATABASE_URL=postgresql://user:password@localhost:5432/lottery_db?schema=public```### 3️⃣ 运行迁移和初始化```bash# 生成 Prisma 客户端npm run db:generate# 创建所有表npm run db:migrate# 填充初始数据npm run db:seed```### 4️⃣ 验证数据库```bash# 打开 Prisma Studio（可视化数据库）npm run db:studio```浏览器会打开 http://localhost:5555，可以看到所有数据。### 5️⃣ 启动服务```bashnpm start```---## 逐步迁移指南### 步骤 PostgreSQLbrew services start postgresql# 或创建数据库createdb lottery_db```### 错误 2：权限错误```Error: permission denied```**解决方案**```bash# 重置数据库npm run db:reset# 或创建新用户psql -U postgresCREATE DATABASE lottery_db;CREATE USER lottery WITH PASSWORD 'password';GRANT ALL PRIVILEGES ON DATABASE lottery_db TO lottery;```### 错误 3：Prisma 客户端未生成```Error: Cannot find module '@prisma/client'```**解决方案**```bashnpm install @prisma/clientnpm run db:generate```### 错误 4：迁移冲突```Error: Migration already exists```**解决方案**```bash# 查看迁移状态npx prisma migrate status# 重置（警告：删除所有数据）npm run db:reset```---## 数据库操作命令### 查看所有表```bashnpm run db:studio```### 手动运行 SQL```bash# PostgreSQLpsql -U user -d lottery_db# MySQLmysql -u user -p
