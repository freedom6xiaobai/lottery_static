# 🗄️ 数据库集成指南## 概述项目已集成 Prisma ORM，支持 PostgreSQL 和 MySQL 两种数据库。本指南说明如何配置和使用数据库。---## 📦 已安装的包```json{  "@prisma/client": "^7.5.0",      // Prisma 客户端  "prisma": "^7.5.0",              // Prisma CLI  "bcryptjs": "^3.0.3",            // 密码加密  "dotenv": "^16.x"                // 环境变量}```---## 🚀 快速开始### 第一步：配置数据库连接编辑 `.env` 文件：**PostgreSQL（推荐）**```envDATABASE_URL=postgresql://user:password@localhost:5432/lottery_db?schema=public```**MySQL**```envDATABASE_URL=mysql://root:password@localhost:3306/lottery_db```### 第二步：创建数据库迁移```bashnpm run db:generate        # 生成 Prisma 客户端npm run db:migrate         # 运行迁移，创建所有表npm run db:seed            # 填充初始数据```### 第三步：启动服务```bashnpm start                  # 使用生产模式npm run dev               # 使用开发模式（带自动重启）```---## 📊 数据库模型### User（用户）```sql- id (primary key)- username (unique)- password (hashed)- email- status (active/inactive)- createdAt- updatedAt```### Role（角色）```sql- id (primary key)- name (unique)- description- createdAt- updatedAt```### Permission（权限）```sql- id (primary key)- name (unique)- description- createdAt- updatedAt```### Menu（菜单）```sql- id (primary key)- name- path- icon- component- parentId (自我关联)- order- createdAt- updatedAt```### 关联表- **UserRole** - User 和 Role 多对多- **RolePermission** - Role 和 Permission 多对多- **RoleMenu** - Role 和 Menu 多对多---## 🔧 常用命令### 数据库操作```bash# 生成 Prisma 客户端npm run db:generate# 创建新的迁移npm run db:migrate# 填充初始数据npm run db:seed# 打开 Prisma Studio（数据库 GUI）npm run db:studio# 重置数据库（删除所有数据）npm run db:reset# 推送模型到数据库（无需迁移文件）npm run db:push```### 开发模式```bash# 使用 nodemon 自动重启npm run dev# 正常启动npm start```---## 📝 更新服务以使用数据库### 原有的内存版本```javascript// services/authService.js - 1：数据库连接失败**症状**```Error: Could not connect to database```**解决方案**1. 检查 `.env` 中的 `DATABASE_URL`2. 确认数据库服务已启动3. 验证用户名和密码是否正确### 问题 2：Prisma 客户端生成失败**症状**```Error: Failed to generate Prisma Client```**解决方案**```bashnpm install @prisma/clientnpm run db:generate```### 问题 3：迁移失败**症状**```Error: Migration failed```**解决方案**```bash# 查看迁移历史npx prisma migrate status# 回滚最后一个迁移npx prisma migrate resolve --rolled-back <migration_name># 重试迁移npm run db:migrate```### 问题 4：权限错误**症状**```Error: permission denied for schema public```**解决方案**```sql-- 以超级用户身份运行 SQLGRANT ALL PRIVILEGES ON SCHEMA public TO your_user;GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO your_user;```---## 🌐 PostgreSQL 本地设置### macOS（使用 Homebrew）```bash# 安装 PostgreSQLbrew install postgresql# 启动服务brew services start postgresql# 创建数据库createdb lottery_db# 连接数据库psql -d lottery_db# 创建用户CREATE USER lottery WITH PASSWORD 'password';ALTER ROLE lottery CREATEDB;GRANT ALL PRIVILEGES ON DATABASE lottery_db TO lottery;```### 连接字符串```postgresql://lottery:password@localhost:5432/lottery_db?schema=public```---## 🗄️ MySQL 本地设置### macOS（使用 Homebrew）```bash# 安装 MySQLbrew install mysql# 启动服务brew services start mysql# 初始化mysql_secure_installation# 创建数据库mysql -u root -pCREATE DATABASE lottery_db;CREATE USER 'lottery'@'localhost' IDENTIFIED BY 'password';GRANT ALL PRIVILEGES ON lottery_db.* TO 'lottery'@'localhost';FLUSH PRIVILEGES;```### 连接字符串```mysql://lottery:password@localhost:3306/lottery_db```---## 📋 数据库操作检查清单- [ ] 安装 Prisma 依赖- [ ] 配置 `.env` 中的 `DATABASE_URL`- [ ] 创建数据库- [ ] 运行迁移：`npm run db:migrate`- [ ]
