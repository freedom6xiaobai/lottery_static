# lottery_static## 项目简介这是一个基于 Node.js 和 Express 的 RBAC（基于角色的访问控制）系统，使用 Prisma 作为 ORM 工具，连接到 MySQL 数据库。该系统支持用户注册、登录、角色管理、权限分配等功能，适用于需要细粒度权限控制的应用场景。## 主要功能- 用户注册和登录- 角色管理（创建、更新、删除角色）- 权限管理（创建、更新、删除权限）- 用户角色分配- 基于角色的访问控制- JWT 认证- 权限验证中间件- 菜单管理（可选）- 日志记录（可选）- 单元测试（可选）- API 文档（可选）- 数据迁移和种子脚本（可选）- 前端集成（可选）- 国际化支持（可选）- 性能优化（可选）- 安全措施（可选）- 持续集成和部署（可选）- 监控和日志分析（可选）- 多数据库支持（可选）- 代码质量工具（可选）- Docker 支持（可选）- 其他功能（根据需求添加）- ## 项目结构- ```plaintext- lottery_static/- ├── src/- │   ├── controllers/ 控制器层，处理请求和响应- │   ├── middleware/ auth 和 permission 中间件- │   ├── routes/ 路由层（auth、user、role、menu）)- │   ├── services/ 业务逻辑层，处理具体的业务逻辑- │   ├── utils/ 工具函数，如 JWT 相关函数- │   └── app.js 应用主文件- ├── .env.example 数据库配置- ├── package.json 完整依赖配置- └── README.md- ```## 安装和运行1. 克隆仓库：2. ```bash    git clone    cd lottery_static    ```3. 安装依赖：4. ```bash    npm install    ```5. 配置环境变量：- 创建 `.env` 文件，参考 `.env.example`，设置数据库连接字符串等配置6. 运行数据库迁移：```bashnpx prisma migrate dev --name init```7. 启动服务器：```bash npm start```8. 访问 API：9. ```bash    http://localhost:3000/api    ```   ## 贡献欢迎贡献！请提交 pull request 或打开 issue 讨论改进建议。## 许可证MIT License# 部署vercel和supabasehttps://vercel.com/freedom6xiaobais-projectshttps://supabase.com/dashboard/project/ydvpisusqbzftbtugkba
